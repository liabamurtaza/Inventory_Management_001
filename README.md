## Overview

A console-based Inventory Management System written entirely in x86 Assembly Language using MASM 6.15 and the Irvine32 library. The system allows users to manage products and record sales through a color-coded interactive menu, with persistent file storage via a binary `.dat` file.

---

## Features

| # | Feature | Description |
|---|---------|-------------|
| 1 | Add Product | Add a new item with ID, Name, Quantity, and Price |
| 2 | View Products | Display all items in a formatted table |
| 3 | Record a Sale | Log a sale and update stock quantity |
| 4 | Update Product | Edit an existing product's details |
| 5 | Delete Product | Remove a product by ID |
| 6 | Exit | Save all data and exit gracefully |

Data is automatically **loaded on startup** and **saved on exit** to `data\inventory.dat`.

## Build Instructions

### 1. Clone the repository
```bash
git clone <repo-url>
cd Inventory_Management_001
```

### 2. Build
Open **Command Prompt** (not PowerShell) and run:
```bat
build.bat
```
This will:
- Assemble all 7 `.asm` source files into `obj\`
- Link them into `main.exe` using the VS2022 linker

### 3. Run
```bat
main.exe
```

> **Note:** Run from **Command Prompt** in the project root directory. Do not double-click the exe — it needs the working directory to locate `data\inventory.dat`.

## How It Works

1. On launch, `LoadData` reads `data\inventory.dat` into the in-memory product array.
2. The menu loops, dispatching to the appropriate handler based on user input.
3. On exit (option 6), `SaveData` writes the entire array back to disk.
4. All I/O uses Irvine32 procedures (`ReadInt`, `ReadString`, `WriteString`, `SetTextColor`).
5. The product record is a fixed-size struct: `ID (DWORD)` + `Name (20 bytes)` + `Quantity (DWORD)` + `Price (DWORD)`.

## Notes

- Build must use **Command Prompt**, not PowerShell or Git Bash.
- `LINK32.EXE` bundled with MASM 6.15 produces 16-bit output on Windows 10/11 — the VS2022 linker is required.
- `SET LINK=` in the batch environment must not be set, as the VS2022 linker reads it as a response file variable.
